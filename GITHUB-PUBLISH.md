# Первая публикация на GitHub

## 1. Подготовить репозиторий

1. Создай новый репозиторий на GitHub.
2. Назови его понятно и уникально.

Рекомендуемые варианты:

- `zapretmanager-desktop`
- `zapretmanager-wpf`
- `zapretmanager-flowseal`

## 2. Что загрузить в репозиторий

В репозиторий загружаются:

- исходный код программы
- `README.md`
- иконки и ресурсы
- `.gitignore`

Не нужно загружать:

- `bin`
- `obj`
- `IntermediateBuilds`
- `Release`

## 3. Что добавить перед первым релизом

- 2–4 хороших скриншота интерфейса
- ссылку на GitHub автора в окне `О программе`
- краткое описание релиза `1.0`

## 4. Как оформить релиз

Для первой публичной версии:

- Tag: `v1.0`
- Title: `ZapretManager 1.0`

В описание релиза можно вынести:

- установка, обновление и откат сборки
- проверка конфигов
- подробные результаты проверки
- DNS и DNS-диагностика
- служба Windows, tray и темы

## 5. Как вести версии дальше

Публичные версии:

- `1.0`
- `1.0.1`
- `1.1`
- `1.1.1`

Внутренние рабочие сборки:

- `1.0-dev01`
- `1.0-dev02`
- `1.0-dev03`
